模型配置简明教程

面向 New API 管理员：把渠道、模型、端点、映射、计费配成一套稳定规则，减少调用失败和日志混乱。

01

先记住一个标准

对外模型名、上游模型名、调用端点、计费规则要各司其职，不要混在参数覆盖里。

渠道负责接上游

渠道里配置供应商类型、Base URL、Key、分组和可用模型。这里的模型名建议使用平台对外暴露的稳定名称。

Channel Base URL API Key

映射负责改上游名

如果对外叫 doubao-seedance-2.0，上游实际叫 doubao-seedance-2-0-260128，写到模型映射，不要写到参数覆盖。

Model Mapping 不要硬改 model

模型页负责能力

模型管理里确认端点类型、可见分组、计费方式和价格。缺价格时容易出现扣费规则不存在或模型列表展示异常。

Endpoint Pricing Group
参数覆盖只放通用、稳定、不会改变用户意图的参数。 不建议在参数覆盖里固定 prompt、content、model、用户素材地址。这样会导致所有请求都被改成同一个内容，日志也会变得难判断。
02

管理员配置流程

每加一个模型都按这五步走，能覆盖标准大模型、图片、视频和语音。

STEP 1 新建渠道 选择正确渠道类型，填写 Base URL、Key、分组。
STEP 2 填写模型列表 只填对外模型名，多个模型用英文逗号分隔。
STEP 3 设置模型映射 只有对外名和上游名不一致时才需要映射。
STEP 4 补模型元数据 在模型管理里确认端点、供应商、分组、状态。
STEP 5 补计费规则 聊天多用倍率，图片、视频、语音多用按次或按秒。
03

不同模型怎么配

重点是选对端点类型。端点不对时，即使模型名存在，也会调用失败。

类型 模型页端点 用户调用接口 配置要点 常见错误
标准大模型
聊天、工具调用、文本生成		 openai 或供应商对应聊天端点 POST /v1/chat/completions 模型列表填对外名；如上游名不同，用模型映射；计费通常用模型倍率。 把聊天模型拿去调 /v1/responses，但当前渠道适配器不支持。
文生图
文本生成图片		 image-generation POST /v1/images/generations 请求必须有 prompt 或供应商要求的文本字段；按次计费时要补模型价格。 模型价格缺失，或把固定 prompt 写进参数覆盖。
图生图
参考图改图		 image-generation POST /v1/images/generations 请求必须带 image、reference 或平台支持的上传字段。 只传 prompt 不传图片，日志里出现 Field required。
文生视频
文本生成视频		 openai-video POST /v1/videos 请求通常要有 prompt 或 content；模型映射放上游真实模型名。 把视频模型发到 /v1/chat/completions 或参数覆盖固定 content。
语音
TTS、声音复刻		 audio-speech POST /v1/audio/speech 普通 TTS 需要 input；复刻类还要参考音频字段。 只传文本不传参考音频，或价格未配置。

推荐的模型映射写法

对外名保持简洁稳定，上游版本号放映射。这样用户不用跟着上游版本号频繁改代码。

{
  "doubao-seedance-2.0": "doubao-seedance-2-0-260128",
  "doubao-seedance-1.5": "doubao-seedance-1-5-pro-251215",
  "doubao-seedream-4-5": "doubao-seedream-4-5-251128"
}

参数覆盖的边界

不要替用户填写业务输入。Doubao Seedream 4.5 的 response_format 不建议做通道级覆盖，保持为空最稳；必须覆盖时只覆盖确认被上游接受的非业务字段。

{}
04

AIPDD 模型配置参考

AIPDD 是任务型模型，不是聊天模型。不要用 /v1/chat/completions 调 AIPDD 图片、视频、语音模型。

能力 模型名 接口 必填输入 计费建议
文生图 aipdd-flux-gguf-t2i /v1/images/generations prompt、text 或 input 按次；即使免费也建议价格填 0
图生图 aipdd-flux-gguf /v1/images/generations image，可选 prompt 按次
图生视频 aipdd-wan2.2-wanx /v1/videos image、prompt，可选 duration 按秒；duration 通常为 5 或 10
视频主体替换 aipdd-wan2.2-animater /v1/videos load_video、prompt、negative_prompt 按次
动作迁移 aipdd-mimic-motion /v1/videos motion_video、appearance_image 按次
对口型视频 aipdd-latentsync-1.5 /v1/videos video、LoadAudio 按次
声音复刻 aipdd-indextts /v1/audio/speech input 或 text，以及 audio 按次

AIPDD 渠道

渠道类型选择 AIPDD，Base URL 使用上游 AIPDD 地址，模型列表填完整的 AIPDD 模型名。通常不需要模型映射。

AIPDD 价格

按次模型要在模型价格里出现。免费模型也建议显式填 0，避免列表、计费和日志判断不一致。

AIPDD 测试

任务型模型的自动测试可能显示未测试或响应时间为 0。重点看真实任务提交、任务查询和日志结果。

05

用户调用示例

下面只展示请求形态。管理员配置正确后，用户只需要使用平台发放的 Token。

标准大模型

curl "$BASE_URL/v1/chat/completions" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-v4-pro",
    "messages": [
      { "role": "user", "content": "写一个三行产品文案" }
    ]
  }'

文生图

curl "$BASE_URL/v1/images/generations" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "aipdd-flux-gguf-t2i",
    "prompt": "高级感产品摄影，柔和棚拍灯光"
  }'

图生图

curl "$BASE_URL/v1/images/generations" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "aipdd-flux-gguf",
    "image": "https://example.com/input.png",
    "prompt": "保留主体，改成电影海报风格"
  }'

文生视频

curl "$BASE_URL/v1/videos" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "doubao-seedance-2.0",
    "prompt": "城市夜景航拍，镜头缓慢前进，电影感"
  }'

AIPDD 图生视频

curl "$BASE_URL/v1/videos" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "aipdd-wan2.2-wanx",
    "image": "https://example.com/input.png",
    "prompt": "镜头缓慢推进，画面稳定",
    "duration": 5
  }'

语音复刻

curl "$BASE_URL/v1/audio/speech" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "aipdd-indextts",
    "input": "这里填写需要合成的文本",
    "audio": "https://example.com/reference.wav"
  }'

异步任务查询

图片、视频、语音任务提交后通常先返回 task_id。按对应类型查询，不要跨接口查询。

# 图片
curl "$BASE_URL/v1/images/generations/task_xxx" \
  -H "Authorization: Bearer $TOKEN"

# 视频
curl "$BASE_URL/v1/videos/task_xxx" \
  -H "Authorization: Bearer $TOKEN"

# 语音
curl "$BASE_URL/v1/audio/speech/task_xxx" \
  -H "Authorization: Bearer $TOKEN"
06

上线前检查和日志排错

出现调用问题时，不要先改参数覆盖，先按清单确认模型链路完整。

  • 渠道已启用，分组包含用户所在分组。
  • 渠道模型列表包含对外模型名，多个模型用英文逗号。
  • 上游真实模型名只写在模型映射里。
  • 模型管理里模型状态为启用，端点类型匹配调用接口。
  • 按次、按秒模型已有模型价格；免费也显式填 0。
  • 参数覆盖没有固定 prompt、content、image、audio、video。
  • 用户调用接口正确：聊天、图片、视频、语音不要混用。
  • 测试时使用普通用户 Token，不使用管理员密码或上游 Key。
日志现象 优先检查 处理方式
model_not_found 渠道模型列表、用户分组、模型端点 把模型加到渠道和模型管理；确认用户调用的是对应端点。
扣费规则不存在 模型价格或计费类型 按次模型补 ModelPrice；倍率计费模型补倍率。
Field required 该模型必填字段 图生图补图片，复刻语音补参考音频，AIPDD 视频按模型补素材字段。
not implemented 接口是否被渠道适配器支持 例如 DeepSeek 不一定支持 /v1/responses，改用 /v1/chat/completions。
response_format 类型错误 参数覆盖格式 删除通道级覆盖。Doubao Seedream 4.5 不要把 response_format 覆盖成对象。
